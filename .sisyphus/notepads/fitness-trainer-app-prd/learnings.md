### 핵심 비즈니스 규칙
- 트레이너:회원 = 1:N (회원은 한 트레이너에게만)
- 예약 상태: 대기중 → 승인됨/거절됨 → 완료/취소됨
- 매뉴얼: 전체 공개 (모든 사용자 열람)
- 운동 프로그램: 자유 텍스트 (구조화 X)
- 채팅 파일: 이미지 10MB, 파일 50MB 제한
- 연결 해제 후 데이터: 아카이브 (삭제 X)

### 문서 구조 규칙
- 각 기능 섹션 필수 하위 섹션: 개요 / 사용자 흐름 / 화면 설명(UI 요소 목록) / 비즈니스 규칙 / 엣지 케이스 / 범위 외
- 언어: 한국어 전용
- DB/API/기술 구현 세부사항 절대 금지

- 카카오 OAuth 기반 인증 및 역할 고정 시스템.
- 인앱 알림 전용 시스템 (OS 푸시 제외).

## 문서 작성 규칙
- UI 설계자가 활용 가능하도록 화면 단위 UI 요소 리스트 포함.
- 기술 용어(ERD, 스키마 등) 배제하고 비즈니스 및 사용자 흐름에 집중.
- 섹션별 6대 구성 요소(개요, 사용자 흐름, 화면 설명, 비즈니스 규칙, 엣지 케이스, 범위 외) 준수.

## 특이사항
- 연결 해제 후에도 데이터는 아카이브 보관(상대방 접근만 차단).
