## 7개 화면 상세

### 화면 1: 로그인 (랜딩)
- PT 매니저 로고 (파란 덤벨 아이콘 + 앱 이름)
- 부제: "체계적인 PT 일정 및 회원 관리 솔루션"
- 카카오 로그인 버튼 (노란색 #FEE500)
- "또는" 구분선
- 이메일 로그인 버튼 (흰색+회색 테두리)
- 하단: 이용약관 + 개인정보처리방침 링크

### 화면 2: 역할 선택 (온보딩 Step 1/3)
- 뒤로가기 + 3단계 프로그레스바 (1번 활성)
- "어떤 역할로 사용하시겠어요?" 제목
- 역할 변경 어려움 안내 문구
- 트레이너 카드 (덤벨 아이콘, 파란)
- 회원 카드 (사람 아이콘, 회색)
- "다음 →" 버튼

### 화면 3: 트레이너 프로필 (온보딩 Step 2/3)
- 프로그레스바 (2번 활성)
- 프로필 사진 업로드 (점선 원형 + 카메라 아이콘)
- 이름(활동명) 입력
- 전문 분야 2x2 그리드 (재활/교정, 근력 증가, 다이어트/식단, 스포츠 퍼포먼스) - 중복선택
- "완료 ✓" 버튼

### 화면 4: 회원 프로필 (온보딩)
- 프로필 사진 업로드
- 기본 정보: 이름, 연락처
- 신체 정보: 나이(세), 키(cm), 몸무게(kg) — 3열
- 운동 목표 2x2: 체중 감량, 근력 증가, 체력 증진, 바디 프로필
- 부상 및 특이사항 텍스트 영역
- "작성 완료 ✓" 버튼

### 화면 5: PT 트레이너 찾기 (검색)
- 검색바 (이름 검색) — PRD 기준: 지역 검색 제외
- 필터 아이콘 (전문분야 필터)
- 트레이너 카드 리스트 (사진+이름+전문분야) — PRD 기준: 지역/경력/모집마감 제외
- "연동 요청하기" / "프로필 보기" 버튼
- 하단 네비게이션 바 (홈/검색/일정/내정보)

### 화면 6: 초대 코드 관리 (트레이너)
- QR 아이콘 + 안내 문구
- 초대 코드 카드 (코드 표시 + 파란 밑줄)
- 코드 복사 / 링크 공유 버튼
- 카카오톡 초대장 보내기 버튼 (노란색)
- 최근 연결된 회원 목록

### 화면 7: 초대 코드 입력 (회원)
- 안내 아이콘 + 설명 텍스트
- 6자리 코드 입력 필드 (개별 박스)
- "코드 확인" 버튼
- "연결 확정" 버튼 (파란색)

## Scope Boundaries
- INCLUDE: 프로젝트 초기화, 7개 화면 Vue 컴포넌트, 라우팅, 디자인 시스템 (CSS 변수), 기본 인터랙션
- INCLUDE: 하단 네비게이션 바 (공통 레이아웃, 로그인/온보딩에서는 숨김)
- INCLUDE: 카드 선택 토글, 버튼 클릭 라우트 이동, 입력필드 포커스 스타일
- EXCLUDE: 백엔드 API 연동, 카카오 OAuth 실제 연동, 실제 데이터 저장, 다른 화면
- EXCLUDE: 다크 모드, i18n, 애니메이션, 로딩 스피너, 에러 페이지, PWA, SEO

## Metis 검토 결과 — 갭 해결
- **이메일 로그인 후속 화면**: 버튼 UI만 표시, 클릭 시 미구현 alert 또는 라우트 없음 (기본값 적용)
- **프로그레스바 3단계**: 역할선택(1/3) → 프로필입력(2/3) → 완료후 자동이동(3단계 없음). 프로그레스바 유지, 3번째 화면 없음.
- **회원 운동목표**: UI 목업 기준 2x2 칩 선택 (PRD 텍스트 영역과 다름 — 목업 우선)
- **폰트**: 시스템 폰트 fallback 전략 (웹폰트 번들 제외)
- **하단 네비**: 공통 레이아웃으로 구현, 로그인/온보딩에서 숨김
- **인터랙션**: 기본 인터랙션 포함 (카드 선택 토글, 라우트 이동, 포커스 스타일)
