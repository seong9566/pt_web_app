# 회원 가능 시간 등록 UX 개선 제안

## 1. 현재 방식의 문제점 (As-Is)
- **화면 이동의 번거로움**: '일정' 화면에서 버튼을 클릭하여 별도의 '가능 시간 등록' 페이지로 이동해야 함.
- **맥락 부재**: 등록 화면에서는 내 기존 수업 일정을 볼 수 없어, 비어있는 시간을 확인하기 위해 화면을 왔다 갔다 해야 함.
- **주간 흐름 파악의 어려움**: 요일별 탭 방식으로 되어 있어 한 주의 시간 흐름을 한눈에 보기가 어려움.

## 2. 제안하는 방식 (To-Be: "달력에서 바로 선택")
별도의 페이지 이동 없이 **내 일정(주간 캘린더)** 화면 내에서 직접 등록 모드를 활성화하여 시간을 선택하는 방식입니다.

### 핵심 기능 및 경험
- **직관적인 시간 선택**: 캘린더의 빈 슬롯을 터치하면 즉시 '선호 시간'으로 등록/해제됨.
- **수업 일정 연동**: 이미 등록된 내 PT 수업(노란색 블록)과 트레이너의 휴무(빗금 처리)를 보면서 일정을 짤 수 있음.
- **편집 모드 도입**: 실수로 인한 등록을 방지하기 위해 '가능 시간 등록' 모드(편집 모드)를 활성화했을 때만 슬롯 선택이 가능함.
- **간편한 저장 및 취소**: 하단 바(Bottom Bar)를 통해 현재 선택된 개수를 확인하고 즉시 저장하거나 취소를 할 수 있음.

## 3. 구현 계획 요약
1. **캘린더 컴포넌트 확장**: `AppWeeklyCalendar`에 `selectionMode`와 `preferredSlots` 속성 추가.
2. **선택 로직 구현**: 빈 슬롯 클릭 시 상위 컴포넌트로 데이터 전달 및 시각적 피드백(파란색 블록 등) 제공.
3. **상태 관리**: `MemberScheduleView`에서 임시 선택된 슬롯 정보를 관리하고 저장 시 서버로 전송.
4. **기존 방식 대체**: 현재의 별도 페이지(`AvailabilityRegistrationView.vue`) 방식을 제거하거나 이 새로운 방식으로 통합.

---
이 개선안을 통해 회원은 더욱 빠르고 정확하게 본인의 선호 시간을 트레이너에게 전달할 수 있게 됩니다.
