# 헬스 트레이너 앱 — PRD 개정안 (일정 예약 구조 개선)

## 개정 목적

기존의 **회원 예약 요청 → 트레이너 승인/거절** 구조는 다음과 같은 문제가 있습니다.

1. 트레이너가 예약을 거절할 경우 회원이 심리적으로 거절당한 느낌을 받을 수 있습니다.
2. 실제 PT 운영 방식은 회원 자율 예약보다 트레이너 주도의 일정 조율에 가깝습니다.
3. 인기 시간대에 회원 간 선착순 경쟁이 발생하여 서비스 경험이 저하될 수 있습니다.

이에 따라 본 문서는 기존 예약 중심 구조를 **트레이너 주도 일정 조율 및 회원 확정 구조**로 개편하기 위한 PRD 문구를 정의합니다.

---

## 변경 요약

### 기존 구조

- 회원이 가능한 시간을 보고 직접 예약 요청
- 트레이너가 승인 또는 거절
- 동일 시간대에 대해 회원 간 선착순 경쟁 가능

### 변경 구조

- 회원은 자신의 가능한 시간 또는 선호 시간대를 전달
- 트레이너가 회원별 스케줄을 고려하여 시간 후보를 제안하거나 일정을 직접 등록
- 회원은 제안된 시간 중 하나를 확정하거나 재조율을 요청
- 공개 슬롯 선점 구조를 제거하여 회원 간 경쟁을 최소화

## 상태값 개편

### 기존 상태

- 대기중
- 승인됨
- 거절됨
- 취소됨
- 완료

### 변경 상태

- 가능 시간 제출됨
- 조율 중
- 시간 제안됨
- 확정됨
- 재조율 필요
- 취소됨
- 완료

### 상태 정의

- **가능 시간 제출됨**: 회원이 자신의 가능한 시간 또는 선호 시간대를 전달한 상태
- **조율 중**: 트레이너가 회원의 요청과 전체 스케줄을 검토 중인 상태
- **시간 제안됨**: 트레이너가 특정 회원에게 후보 시간 1개 이상을 전달한 상태
- **확정됨**: 회원이 제안 시간 중 하나를 선택했거나 트레이너가 직접 등록한 일정을 회원이 확인한 상태
- **재조율 필요**: 기존 제안 시간으로 진행이 어려워 새로운 시간 조율이 필요한 상태
- **취소됨**: 확정된 일정이 취소된 상태
- **완료**: 일정 종료 시점이 지나 자동 완료된 상태

> 주의: 회원 경험 보호를 위해 **거절됨** 상태는 사용하지 않습니다.

## 8. 트레이너 근무시간 및 일정 조율 단위 설정

### 개요

트레이너가 회원과 일정을 조율할 수 있도록 기본 근무 가능 시간과 세션 단위를 설정하는 기능입니다. 회원은 이 정보를 직접 선점하는 것이 아니라, 트레이너가 조율 과정에서 참고하여 일정 후보를 제안하거나 일정을 생성합니다.

### 사용자 흐름

- **근무시간 설정**: 트레이너가 설정 메뉴에 진입합니다. 근무시간 설정을 선택하고 활성화할 요일을 정합니다. 각 요일별로 시작 시간과 종료 시간을 입력하고 PT 세션 단위를 선택한 뒤 저장합니다.
- **휴무일 설정**: 트레이너가 캘린더 화면에서 특정 날짜를 선택합니다. 휴무 설정 버튼을 눌러 해당 날짜의 일정 생성을 제한합니다.
- **조율 기준 활용**: 트레이너는 회원의 선호 가능 시간과 본인의 근무시간을 함께 참고하여 일정 후보를 제안합니다.

### 화면 설명

1. **근무시간 설정 화면**

   - 타이틀: "근무시간 설정"
   - 요일 선택: 일, 월, 화, 수, 목, 금, 토 토글 스위치
   - 시간 입력: 요일별 시작/종료 시간 선택 드롭다운 (예: 09:00 ~ 18:00)
   - 단위 선택: 30분, 1시간, 1시간 30분, 2시간 라디오 버튼
   - 버튼: [저장]

2. **휴무일 설정 화면**

   - 뷰: 월간 캘린더
   - 동작: 날짜 선택 시 하단에 [휴무 설정] 또는 [휴무 해제] 토글 표시
   - 표시: 휴무로 지정된 날짜는 캘린더에서 별도 색상으로 강조
   - 버튼: [변경 사항 저장]

### 비즈니스 규칙

- 근무시간은 트레이너가 일정 조율 시 참고하는 기본 운영 시간입니다.
- 회원은 근무시간을 기준으로 직접 슬롯을 선점하지 않습니다.
- 세션 단위는 트레이너가 일정 후보를 생성하거나 직접 일정을 등록할 때 적용됩니다.
- 근무시간을 하나도 설정하지 않은 경우에도 트레이너는 수동으로 일정을 생성할 수 있으나, 자동 후보 생성 기능은 제한될 수 있습니다.
- 이미 확정된 일정이 있는 상태에서 근무시간을 변경하더라도 기존 일정은 자동 취소되지 않으며, 이후 조율 건에만 반영됩니다.
- 기본적으로 주간 단위로 반복 설정되며, 휴무 설정을 통해 특정 날짜만 예외 처리할 수 있습니다.

### 엣지 케이스

- **중복 일정 충돌**: 근무시간 변경 후 기존 확정 일정과 충돌 가능성이 생기면 트레이너에게 경고를 표시하되, 기존 일정은 유지합니다.
- **시간 설정 오류**: 종료 시간이 시작 시간보다 빠르거나 같으면 "유효하지 않은 시간 범위입니다"라는 안내를 표시합니다.

### 범위 외

- 회원이 직접 실시간 빈 슬롯을 보고 선점 예약하는 기능
- 동일 시간에 여러 명을 받는 오버부킹 설정
- AI 자동 최적 스케줄 추천 기능

## 9. PT 일정 조율 및 확정 시스템

### 개요

회원이 자신의 가능한 시간 또는 선호 시간대를 전달하고, 트레이너가 전체 회원 스케줄을 고려하여 시간 후보를 제안하거나 일정을 직접 생성하는 시스템입니다. 회원은 제안된 일정 중 하나를 확정하거나 재조율을 요청할 수 있습니다.

기존의 예약 요청/승인/거절 구조 대신, **조율 중심의 일정 확정 구조**를 적용합니다.

### 사용자 흐름

#### 회원

- 홈 또는 일정 메뉴에서 [가능 시간 전달] 버튼을 누릅니다.
- 이번 주 또는 특정 기간 동안 가능한 시간대를 선택합니다.
- 필요 시 메모를 입력합니다. (예: "화/목 8시 이후 가능")
- 트레이너가 보낸 일정 후보를 확인합니다.
- 제안된 시간 중 하나를 선택하여 확정하거나, [다른 시간 요청]을 눌러 재조율을 요청합니다.

#### 트레이너

- 회원별 가능 시간 제출 내역을 확인합니다.
- 본인의 근무시간, 휴무일, 기존 확정 일정, 다른 회원 일정 등을 고려하여 시간 후보를 1개 이상 제안합니다.
- 필요 시 특정 회원의 일정을 직접 생성하여 확정 대기 상태로 전달할 수 있습니다.
- 회원의 응답을 확인한 뒤 일정을 확정 상태로 전환합니다.

### 화면 설명

1. **가능 시간 전달 화면(회원용)**

   - 상단: 기간 선택 (예: 이번 주, 다음 주)
   - 본문: 시간대 선택 UI (요일/시간대 기반 다중 선택)
   - 추가 입력: 선호 시간 메모 입력란
   - 버튼: [전달하기]

2. **일정 후보 확인 화면(회원용)**

   - 정보: 트레이너 이름, 제안된 날짜/시간 목록, 세션 단위
   - 버튼: 각 시간 후보별 [이 시간으로 확정]
   - 보조 버튼: [다른 시간 요청]

3. **회원별 일정 조율 화면(트레이너용)**

   - 상단: 회원 프로필, 잔여 PT 횟수, 최근 일정 요약
   - 중단: 회원이 제출한 가능 시간 및 메모
   - 하단: 일정 후보 추가 UI
   - 버튼: [후보 보내기], [직접 일정 등록]

4. **일정 목록 화면(트레이너용)**

   - 필터: 전체, 조율 중, 시간 제안됨, 확정됨, 완료
   - 리스트: 회원명, 날짜/시간, 상태 배지
   - 정렬: 조율 중 및 응답 대기 항목 우선 노출

5. **재조율 요청 팝업**

   - 입력: 재조율 사유 선택 또는 자유 입력
   - 버튼: [보내기], [닫기]

### 비즈니스 규칙

- 회원은 직접 빈 슬롯을 선점하거나 예약 요청을 생성하지 않습니다.
- 트레이너는 회원별 가능 시간, 본인 근무시간, 기존 일정 현황을 고려하여 시간 후보를 제안합니다.
- 시간 후보는 한 번에 1개 이상 제안할 수 있습니다.
- 회원은 제안된 시간 중 하나만 확정할 수 있습니다.
- 트레이너는 필요한 경우 회원에게 일정을 직접 등록할 수 있으며, 회원은 이를 확인하거나 재조율을 요청할 수 있습니다.
- 일정 조율 과정에서는 **거절** 상태를 사용하지 않고, 필요 시 **재조율 필요** 상태로 처리합니다.
- 확정된 일정은 시작 시간이 지나면 시스템이 자동으로 완료 상태로 변경합니다.
- 잔여 PT 횟수가 부족하더라도 일정 조율 자체는 가능하지만, 트레이너와 회원 모두에게 경고를 표시합니다.

### 엣지 케이스

- **동시 제안 충돌**: 트레이너가 서로 다른 회원에게 동일 시간대를 제안한 경우, 먼저 확정된 일정만 유지하고 나머지는 자동으로 재조율 필요 상태로 전환합니다.
- **응답 지연**: 회원이 일정 후보에 일정 시간 이상 응답하지 않으면 트레이너에게 확인 알림을 보냅니다.
- **휴무일 변경**: 후보 제안 후 해당 날짜가 휴무일로 변경되면, 관련 조율 건은 재조율 필요 상태로 전환됩니다.
- **기존 일정과 충돌하는 직접 등록**: 트레이너가 이미 확정된 시간에 중복 등록하려 하면 저장을 차단합니다.

### 범위 외

- 회원 간 공개 선착순 예약 기능
- 대기 명단 자동 배정 기능
- 기존 확정 일정을 클릭 한 번으로 다른 시간으로 자동 이동시키는 기능

## 10. 캘린더

### 개요

트레이너와 회원이 전체 PT 일정을 달력 형태로 확인하는 기능입니다. 트레이너는 회원별 조율 중 일정과 확정 일정을 함께 관리하고, 회원은 본인의 제안 대기 일정과 확정 일정을 확인합니다.

### 사용자 흐름

- 하단 네비게이션에서 캘린더 탭을 누르면 이번 달 일정이 표시됩니다.
- 특정 날짜를 누르면 하단에 해당 날짜의 조율 중 일정, 제안된 일정, 확정 일정 목록이 표시됩니다.
- 목록에서 항목을 선택하면 일정 상세 화면으로 이동합니다.
- 트레이너는 캘린더 내에서 회원별 조율 상태를 확인하고 시간 후보를 추가 제안하거나 직접 확정 일정을 생성할 수 있습니다.

### 화면 설명

1. **캘린더 메인 화면**

   - 상단: 월 이동 화살표, 현재 연월 표시
   - 전환: 월간, 주간, 일간 뷰 선택 탭
   - 월간 뷰: 날짜 칸 안에 상태별 색상 점(도트) 표시
   - 상태 색상:

     - 조율 중: 노랑
     - 시간 제안됨: 파랑
     - 확정됨: 초록
     - 완료: 회색
     - 휴무일: 배경 음영 처리

2. **주간 및 일간 뷰**

   - 형태: 시간대별 타임라인 방식
   - 표시: 확정 일정은 블록 형태로 노출, 조율 중/제안됨 일정은 보조 카드 또는 별도 레이어로 구분 표시

3. **일별 일정 목록 패널**

   - 리스트: 시간, 상대방 이름, 상태 배지
   - 트레이너 기능: 조율 중인 회원에게 [후보 제안], [직접 등록] 버튼 노출
   - 회원 기능: 시간 제안됨 상태에서 [확정], [재조율 요청] 버튼 노출

### 비즈니스 규칙

- 트레이너는 본인에게 연결된 모든 회원의 일정 조율 현황과 확정 일정을 통합하여 확인합니다.
- 회원은 본인과 관련된 일정만 캘린더에서 볼 수 있습니다.
- 확정 일정과 조율 중 일정을 시각적으로 구분하여 표시해야 합니다.
- 지나간 일정도 완료 및 취소 이력으로 확인할 수 있습니다.
- 휴무일은 일정 생성 불가 일자로 표시되며, 조율 단계에서도 후보 제안 제한 기준으로 활용됩니다.

### 엣지 케이스

- **일정 없음**: 선택한 날짜에 일정이 없으면 "등록된 일정이 없습니다"라는 메시지를 표시합니다.
- **조율 건만 존재**: 확정 일정은 없고 조율 중인 항목만 있는 경우 이를 별도 섹션으로 분리하여 보여줍니다.
- **휴무일 표시**: 휴무일은 캘린더 배경색을 다르게 표시하여 회원에게도 해당 날짜가 일반 운영일이 아님을 알립니다.

### 범위 외

- 외부 캘린더(구글, 애플 등) 연동
- 개인 To-do 기능
- OS 푸시 기반 리마인더 알림

## 최종 방향

본 서비스는 일반적인 공개 예약 플랫폼이 아니라, **트레이너가 회원별 PT 일정을 효율적으로 조율하고 관리하는 운영형 서비스**를 지향합니다. 따라서 예약 선점 구조보다 **트레이너 주도형 일정 제안 및 회원 확정 구조**가 제품 목적과 실제 현장 운영 방식에 더욱 적합합니다.
