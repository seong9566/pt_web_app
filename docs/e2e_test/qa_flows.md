- 작성일: 2026-03-17

- 기준 커밋: `db17829`
- 참고 문서: `docs/e2e_test/bug.md` 버그 리포트, 상태 전이/알림 타입 정리 메모

## 전체 플로우 다이어그램

```text
회원-트레이너 연결 완료
  -> 회원 가용 시간 등록 또는 트레이너 직접 배정 준비
  -> 트레이너가 /trainer/schedule 주간 탭에서 빈 슬롯 선택
  -> 회원 선택 후 일정 배정, 상태: scheduled
  -> 회원은 /member/schedule 에서 일정 확인
     -> 확정, 상태: confirmed
     -> 변경 요청, 상태: change_requested
        -> 트레이너가 /trainer/reservations 에서 재배정
        -> 기존 일정 cancelled, 신규 일정 scheduled
  -> 트레이너 또는 회원이 취소, 상태: cancelled
  -> 수업 완료 처리 시 상태: completed, PT 차감 트리거 실행
  -> 각 단계마다 알림 배지, 알림 목록, 읽음 처리 동작 확인
```

## 상태 전이 매트릭스

| `change_requested` | 트레이너가 재배정 | 기존 일정 `cancelled`, 신규 일정 `scheduled` | 트레이너 | 회원에게 `schedule_reassigned`, 새 일정 노출 |
| `scheduled` | 트레이너 또는 회원이 취소 | `cancelled` | 양측 | 상대방에게 `reservation_cancelled` |

## QA 1. 트레이너 → 회원 일정 배정

### 사전조건
- 트레이너와 회원 연결이 `active` 상태다.
- 트레이너 근무시간 설정이 완료되어 주간 슬롯이 노출된다.
- 같은 시간대에 `scheduled`, `confirmed` 충돌 일정이 없다.

### 단계별 플로우
1. 트레이너 계정으로 로그인한다.
2. `/trainer/schedule` 진입 후 `주간` 탭이 선택된 상태를 확인한다.
3. 빈 슬롯 하나를 탭한다.
4. 회원 선택 바텀시트에서 대상 회원을 선택한다.
5. 배정 완료 토스트를 확인한다.
6. 주간 캘린더에 노란색 일정 블록이 표시되는지 확인한다.

### 검증 항목
- 트레이너 화면에서 새 일정 블록이 즉시 보인다.
- 회원 화면 `/member/schedule` 에 동일 날짜, 동일 시간의 노란색 블록이 보인다.
- 회원 알림 배지가 1 증가한다.
- 회원 알림 목록에 `PT 일정이 배정되었습니다` 제목과 일정 안내 본문이 생성된다.

## QA 2. 회원 일정 확정

### 사전조건
- 상태가 `scheduled` 인 일정이 1건 이상 존재한다.

### 단계별 플로우
1. 회원 계정으로 로그인한다.
2. `/member/schedule` 로 이동한다.
3. 노란색 일정 블록을 탭한다.
4. 상세 바텀시트에서 일정 정보와 액션 버튼을 확인한다.
5. `확정` 버튼을 누른다.
6. 캘린더 블록 색상이 초록색으로 바뀌는지 확인한다.

### 검증 항목
- 회원 화면에서 해당 일정 상태가 `confirmed` 로 반영된다.
- 트레이너 알림 배지가 1 증가한다.
- 트레이너 알림 목록에 제목 `일정 확정`, 본문 `회원이 배정된 일정을 확정했습니다.` 가 표시된다.
- 트레이너 `/trainer/schedule` 에서 동일 일정이 확정 상태 색상으로 보인다.

## QA 3. 회원 일정 변경 요청

### 사전조건
- 상태가 `scheduled` 인 일정이 1건 이상 존재한다.

### 단계별 플로우
1. 회원 계정으로 로그인한다.
2. `/member/schedule` 에서 배정된 일정 블록을 탭한다.
3. 상세 바텀시트에서 `변경 요청` 버튼을 누른다.
4. 변경 사유 입력 필드에 사유를 입력한다.
5. `요청 보내기` 버튼을 누른다.
6. 요청 완료 토스트를 확인한다.

### 검증 항목
- 회원 화면에서 해당 일정이 `change_requested` 상태로 표시된다.
- 트레이너 알림 배지가 1 증가한다.
- 트레이너 알림 목록에 제목 `일정 변경 요청`, 본문 `회원이 일정 변경을 요청했습니다.` 가 표시된다.
- 트레이너 `/trainer/reservations` 의 `변경요청` 탭에 해당 건이 노출된다.
- 에러 케이스로 사유를 비우고 제출하면 얼럿이 표시되고 요청이 생성되지 않는다.

## QA 4. 트레이너 일정 재배정

### 사전조건
- 상태가 `change_requested` 인 일정이 존재한다.
- 재배정할 새 날짜와 시간이 트레이너 근무시간 안이며 충돌이 없다.

### 단계별 플로우
1. 트레이너 계정으로 로그인한다.
2. `/trainer/reservations` 로 이동한다.
3. `변경요청` 탭을 선택한다.
4. 대상 일정의 `재배정` 액션을 누른다.
5. 새 날짜와 시간을 선택한다.
6. 확인 버튼을 눌러 재배정을 완료한다.

### 검증 항목
- 기존 일정은 트레이너 목록에서 활성 일정으로 보이지 않는다.
- 회원 화면에서 기존 일정 블록이 사라진다.
- 회원 화면에 새 날짜, 새 시간의 일정 블록이 새로 표시된다.
- 회원 알림 배지가 1 증가한다.
- 회원 알림 목록에 일정 변경 결과를 안내하는 재배정 알림이 생성된다.

## QA 5. 일정 취소, 트레이너/회원

### 사전조건
- 취소 가능한 `scheduled` 또는 `confirmed` 일정이 존재한다.

### 단계별 플로우
1. 트레이너 취소 흐름을 확인한다.
2. `/trainer/reservations` 에서 대상 일정을 선택한다.
3. `취소` 버튼을 누르고 확인한다.
4. 목록에서 일정이 사라지거나 취소 상태로 이동하는지 확인한다.
5. 회원 취소 흐름을 확인한다.
6. `/member/schedule` 에서 확정 일정을 탭한다.
7. `취소` 버튼을 누르고 확인한다.
8. 회원 캘린더에서 해당 블록이 사라지는지 확인한다.

### 검증 항목
- 트레이너 취소 시 회원에게 `일정 취소` 알림이 발송된다.
- 회원 취소 시 트레이너에게 `일정 취소` 알림이 발송된다.
- 양측 모두 알림 탭에서 취소 시점과 대상 일정을 식별할 수 있다.
- 취소된 일정은 활성 일정 집계에서 제외된다.

## QA 6. 월간 캘린더 날짜 선택, Bug #2 수정 검증

### 사전조건
- 트레이너 스케줄에 여러 날짜의 일정이 존재하면 확인이 쉽다.

### 단계별 플로우
1. 트레이너 계정으로 로그인한다.
2. `/trainer/schedule` 로 이동한다.
3. `월간` 탭을 선택한다.
4. 현재 월에서 일정이 있는 날짜 하나를 탭한다.

### 검증 항목
- 날짜를 탭한 뒤에도 `월간` 탭이 유지된다.
- 자동으로 `주간` 탭으로 전환되지 않는다.
- 아래 일정 목록에는 선택한 날짜의 일정만 필터링되어 표시된다.
- 다른 날짜를 탭하면 목록이 새 선택 날짜 기준으로 갱신된다.

## QA 7. 회원 가용 시간 등록

### 사전조건
- 트레이너와 회원 연결이 완료되어 있다.
- 회원이 가용 시간 등록 가능한 주차를 선택할 수 있다.

### 단계별 플로우
1. 회원 계정으로 로그인한다.
2. `/member/availability` 로 이동한다.
3. 등록할 주차를 선택한다.
4. 각 요일을 탭하며 가능한 시간 슬롯을 선택한다.
5. 필요 시 메모를 입력한다.
6. `전달하기` 버튼을 누른다.

### 검증 항목
- 등록 성공 토스트 또는 성공 상태가 보인다.
- 동일 주차 재진입 시 선택한 시간대가 유지된다.
- 트레이너 알림 배지가 1 증가한다.
- 트레이너 알림 목록에 `가능 시간 등록` 알림이 생성된다.

## QA 8. 트레이너 주간 캘린더 가용성 표시, Bug #3 수정 검증

### 사전조건
- QA 7 이 완료되어 회원 가용 시간 데이터가 저장되어 있다.
- 해당 주차를 트레이너 주간 캘린더에서 조회 중이다.

### 단계별 플로우
1. 트레이너 계정으로 로그인한다.
2. `/trainer/schedule` 의 `주간` 탭으로 이동한다.
3. 회원이 가능 시간으로 등록한 날짜와 시간대를 확인한다.
4. 해당 시간 슬롯을 탭해 회원 선택 바텀시트를 연다.

### 검증 항목
- 회원이 등록한 시간대의 빈 셀에 파란색 배경이 표시된다.
- 슬롯 탭 시 해당 회원 옆에 `✅ 가능 시간` 표시가 노출된다.
- 미등록 회원은 가능, 불가, 미확인 상태가 구분되어 보인다.
- 가용성 표시는 기존 예약 블록 표시와 충돌하지 않는다.

## QA 9. 트레이너 가용 시간 현황 히트맵

### 사전조건
- 1명 이상 회원이 가용 시간을 등록했다.
- 최소 1명 이상은 미등록 상태면 배너 검증이 쉬워진다.

### 단계별 플로우
1. 트레이너 계정으로 로그인한다.
2. `/trainer/availability-status` 로 이동한다.
3. 주간 히트맵에서 시간대별 강도를 확인한다.
4. 셀을 탭해 해당 시간대 가능 회원 목록을 확인한다.
5. 미등록 회원 배너가 보이면 선택한다.
6. `등록 요청` 액션을 눌러 리마인더를 보낸다.

### 검증 항목
- 히트맵 색 강도가 등록 인원 수에 따라 달라진다.
- 셀 상세 시트에서 해당 시간대 가능한 회원 목록이 보인다.
- 미등록 회원이 있으면 상단 배너 또는 목록이 노출된다.
- 리마인더 전송 후 회원에게 `가능 시간 등록 요청` 알림이 생성된다.

## QA 10. 알림 시스템

### 사전조건
- 일정 배정, 확정, 변경 요청, 가용 시간 등록 등으로 알림이 누적되어 있다.

### 단계별 플로우
1. 트레이너 또는 회원 계정으로 로그인한다.
2. 상단 벨 아이콘 또는 하단 진입점에서 알림 배지를 확인한다.
3. `/notifications` 로 이동한다.
4. 미읽음 항목과 읽은 항목의 시각 구분을 확인한다.
5. 알림 하나를 탭한다.
6. 읽음 처리와 관련 화면 이동을 확인한다.
7. 다시 알림 목록으로 돌아와 `전체 읽음` 을 실행한다.

### 검증 항목
- 미읽음 개수와 배지 숫자가 일치한다.
- 알림 탭 시 해당 항목이 읽음 처리된다.
- `reservation` 타입은 역할에 따라 `/trainer/reservations` 또는 `/member/schedule` 로 이동한다.
- `전체 읽음` 이후 배지가 0이 되고 미읽음 강조가 사라진다.
- 최근 알림이 최신순으로 정렬된다.
