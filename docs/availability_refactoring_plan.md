# 회원 가능 시간 등록 UX 리팩토링 설계

기존 타일 선택 방식의 한계를 개선하고, 직관적인 캘린더 기반의 가시성 및 조작성을 확보하기 위한 UI/UX 리팩토링 방안입니다.

## 1. 개선 배경 및 목표
현재의 '요일 탭 전환 + 타일형 시간 버튼 선택' 방식은 주 단위로 일정을 파악하거나 연속된 시간을 드래그하여 등록할 수 없기 때문에 입력 피로도가 큽니다.
또한, 메인 캘린더 탭 화면(`/member/schedule`)에서 본인이 비워둔 스케줄 영역을 직관적으로 확인하기 위해 매번 '수정(등록)' 화면으로 전환해야 하는 인지적 불편함이 발생합니다.
본 리팩토링은 메인 화면의 정보 가시성 증대와 시간 선택의 직관성을 높이는 것을 목표로 합니다.

## 2. 화면별 개선 상세 (Proposed Changes)

### [가시성 강화] 메인 스케줄 캘린더 (MemberScheduleView)
회원이 자신의 캘린더 메인 탭에 들어올 때, 이미 시스템에 비워둔 '가능 시간'을 투명도 있는 배경색이나 점선 패턴으로 함께 렌더링합니다.

*   **UI 변경 사항:** 주간 캘린더 컴포넌트(`AppWeeklyCalendar.vue`)에서 회원의 설정 데이터(`available_slots`)를 바탕으로 빈 캘린더 영역 중 내가 열어둔 시간을 은은하게(예: `var(--color-blue-light)` 혹은 빗금 패턴) 표시하여 확정 스케줄과 분리하여 렌더링.
*   **버튼 명칭 변경:** 하단 FAB 버튼의 라벨을 `+ 가능 시간 등록`에서 등록/확인을 포괄할 수 있는 **`⚙️ 가능 시간 관리`** 혹은 **`나의 일정 열어두기`** 등으로 수정.

### [조작성 향상] 가능 시간 등록 화면 (AvailabilityRegistrationView)
요일별 리스트 버튼 뭉치 UI를 전면 제거하고, 구글 캘린더처럼 2D 그리드 레이아웃의 시간표 형태를 도입하여 드래그 앤 드롭으로 시간을 한 번에 색칠할 수 있도록 조작 방식을 변경합니다.

*   **UI 변경 사항:** 
    *   상단 요일 탭과 타일형 버튼 컴포넌트 제거.
    *   기존에 구현된 `AppWeeklyCalendar.vue`의 그리드 레이아웃(월~일 / 06시~22시)을 재활용하여 모달이나 전체 화면에 주간 시간표 노출.
    *   빈 시간표 타일을 꾹 누르고 드래그(Swipe)하거나 탭하여 "연속된 시간"을 한눈에 한 번에 선택(색칠)하는 기능 구현.
*   **루틴 기능 도입 논의 (선택 사항):** 반복되는 주 단위 스케줄(예: 매주 화, 목 19시)의 피로도를 낮추기 위해, "특정 주간"이 아닌 **[나의 기본 가능 루틴]**을 저장하는 메뉴 형태로 진화 고려.
*   **마이크로카피 최적화:** 저장 버튼을 `전달하기` (모호함) 에서 상태 변경을 명확히 하는 **`이대로 저장하기`** 혹은 **`선택 완료`**로 수정.

## 3. 마이그레이션 및 기대 효과 (Verification Plan)
이 개선안이 적용되면 회원들은 트레이너가 보내준 확정 PT 스케줄 외에도, '이번 주에 내가 언제 언제 갈 수 있다고 열어두었는지' 버튼을 누르지 않아도 즉각 인지할 수 있게 됩니다.
드래그 방식의 캘린더 타일 선택기는 모바일 터치 환경에서 반복적인 탭핑을 줄여 UX 피로도를 최대 80% 이상 줄여줄 것입니다.

### 검증 시나리오
1. `member/schedule` 진입 시 FAB 버튼명이 직관적(`가능 시간 관리`)으로 변경되었는지 확인.
2. 메인 캘린더 배경에서 내가 열어둔 스케줄 타임이 어둡지 않은 폰트나 연한 패턴으로 시각화되어 있는지 테스트.
3. FAB 버튼 클릭 후 나오는 그리드(달력 표) 뷰 위에서 손가락 드래그만으로 원하는 3~4시간의 블록이 손쉽게 덩어리로 선택되고, `저장` 버튼이 자연스럽게 연동되는지 확인.
