# 헬스 트레이너 앱 — PRD 개정안 (일정 예약 구조 개선)

## 개정 목적

기존의 **회원 예약 요청 → 트레이너 승인/거절** 구조는 다음과 같은 문제가 있습니다.

1. 트레이너가 예약을 거절할 경우 회원이 심리적으로 거절당한 느낌을 받을 수 있습니다.
2. 실제 PT 운영 방식은 회원 자율 예약보다 트레이너 주도의 일정 배정에 가깝습니다.
3. 인기 시간대에 회원 간 선착순 경쟁이 발생하여 서비스 경험이 저하될 수 있습니다.

이에 따라 본 문서는 기존 예약 중심 구조를 **트레이너 주도 일정 배정 및 회원 확인 구조**로 개편하기 위한 PRD 문구를 정의합니다.

---

## 변경 요약

### 기존 구조

- 회원이 가능한 시간을 보고 직접 예약 요청
- 트레이너가 승인 또는 거절
- 동일 시간대에 대해 회원 간 선착순 경쟁 가능

### 변경 구조

- 회원은 매주 자신의 가능한 시간대를 등록
- 트레이너가 회원별 가능 시간을 확인하고 일정을 직접 배정
- 회원은 배정된 일정을 확인하거나, 사정이 있으면 변경을 요청
- 트레이너가 회원 등록 없이도 직접 일정을 생성할 수 있음 (전화/대면 합의 후)
- 공개 슬롯 선점 구조를 제거하여 회원 간 경쟁 원천 차단
- **거절 상태 자체가 존재하지 않음**

---

## 핵심 플로우

```
[매주] 회원이 이번 주 가능 시간 등록 (요일 × 시간대 다중 선택)
            ↓
       트레이너가 회원별 가능 시간 + 본인 스케줄 확인
            ↓
       트레이너가 빈 슬롯에 회원 배정 (캘린더에서 직접)
            ↓
       회원에게 배정 알림 발송
            ↓
       회원: [확인] 또는 [변경 요청 + 사유]
            ↓
       (변경 요청 시) 트레이너가 다른 시간으로 재배정
```

### 트레이너 직접 배정 (회원 등록 없이)

```
트레이너가 캘린더에서 빈 슬롯 선택
      ↓
회원 목록에서 대상 회원 선택
      ↓
일정 생성 → 회원에게 알림
      ↓
회원: [확인] 또는 [변경 요청]
```

> 회원이 가능 시간을 등록하지 않았더라도, 트레이너가 전화·대면·채팅으로 합의한 뒤 직접 배정할 수 있습니다.

## 상태값 개편

### 기존 상태 (5개)

- 대기중 (pending)
- 승인됨 (approved)
- 거절됨 (rejected)
- 취소됨 (cancelled)
- 완료 (completed)

### 변경 상태 (4개 + 취소)

- 배정됨 (scheduled)
- 확정됨 (confirmed)
- 변경요청 (change_requested)
- 완료 (completed)
- 취소됨 (cancelled)

### 상태 정의

- **배정됨 (scheduled)**: 트레이너가 회원에게 일정을 배정한 상태. 회원의 확인을 기다리는 중.
- **확정됨 (confirmed)**: 회원이 배정된 일정을 [확인]한 상태. PT 진행 예정.
- **변경요청 (change_requested)**: 회원이 배정된 시간에 사정이 있어 변경을 요청한 상태. 트레이너가 다른 시간으로 재배정 필요.
- **완료 (completed)**: 일정 종료 시점이 지나 자동으로 완료된 상태. PT 횟수 자동 차감.
- **취소됨 (cancelled)**: 확정된 일정이 취소된 상태. 트레이너 또는 회원 모두 취소 가능.

> 주의: 회원 경험 보호를 위해 **거절됨** 상태는 사용하지 않습니다. 시간이 맞지 않으면 **변경 요청**으로 처리합니다.

### 상태 전이 다이어그램

```
                    ┌──→ 취소됨 (cancelled)
                    │
배정됨 (scheduled) ─┤──→ 확정됨 (confirmed) ──→ 완료 (completed)
       ↑            │          │
       │            │          └──→ 취소됨 (cancelled)
       │            │
       │            └──→ 변경요청 (change_requested)
       │                        │
       └────────────────────────┘
              트레이너가 시간 수정 후 재배정
```

### 상태 전이 규칙

| 현재 상태 | 다음 상태 | 전이 조건 | 주체 |
|----------|----------|----------|------|
| — | 배정됨 | 트레이너가 일정 생성 | 트레이너 |
| 배정됨 | 확정됨 | 회원이 [확인] 탭 | 회원 |
| 배정됨 | 변경요청 | 회원이 [변경 요청] + 사유 입력 | 회원 |
| 배정됨 | 취소됨 | 트레이너 또는 회원이 취소 | 양쪽 |
| 변경요청 | 배정됨 | 트레이너가 다른 시간으로 재배정 | 트레이너 |
| 확정됨 | 취소됨 | PT 시작 전 취소 | 양쪽 |
| 확정됨 | 완료 | 일정 종료 시간 경과 (자동) | 시스템 |

## 8. 트레이너 근무시간 및 일정 배정 단위 설정

### 개요

트레이너가 일정을 배정할 수 있도록 기본 근무 가능 시간과 세션 단위를 설정하는 기능입니다. 근무시간은 트레이너가 캘린더에서 회원을 배정할 때 기준 시간대로 활용됩니다.

### 사용자 흐름

- **근무시간 설정**: 트레이너가 설정 메뉴에 진입합니다. 근무시간 설정을 선택하고 활성화할 요일을 정합니다. 각 요일별로 시작 시간과 종료 시간을 입력하고 PT 세션 단위를 선택한 뒤 저장합니다.
- **휴무일 설정**: 트레이너가 캘린더 화면에서 특정 날짜를 선택합니다. 휴무 설정 버튼을 눌러 해당 날짜의 일정 생성을 제한합니다.

### 화면 설명

1. **근무시간 설정 화면**

   - 타이틀: "근무시간 설정"
   - 요일 선택: 월, 화, 수, 목, 금, 토, 일 토글 스위치
   - 시간 입력: 요일별 시작/종료 시간 선택 드롭다운 (예: 09:00 ~ 18:00)
   - 단위 선택: 30분, 1시간, 1시간 30분, 2시간 라디오 버튼
   - 버튼: [저장]

2. **휴무일 설정 화면**

   - 뷰: 월간 캘린더
   - 동작: 날짜 선택 시 하단에 [휴무 설정] 또는 [휴무 해제] 토글 표시
   - 표시: 휴무로 지정된 날짜는 캘린더에서 별도 색상으로 강조
   - 버튼: [변경 사항 저장]

### 비즈니스 규칙

- 근무시간은 트레이너가 일정을 배정할 때 슬롯을 생성하는 기준 시간입니다.
- 회원은 근무시간을 직접 조회하거나 슬롯을 선점하지 않습니다.
- 세션 단위는 트레이너가 일정을 배정할 때 블록 크기로 적용됩니다.
- 근무시간을 하나도 설정하지 않은 경우에도 트레이너는 수동으로 일정을 배정할 수 있습니다.
- 이미 확정된 일정이 있는 상태에서 근무시간을 변경하더라도 기존 일정은 자동 취소되지 않으며, 이후 배정 건에만 반영됩니다.
- 기본적으로 주간 단위로 반복 설정되며, 휴무 설정을 통해 특정 날짜만 예외 처리할 수 있습니다.

### 엣지 케이스

- **중복 일정 충돌**: 근무시간 변경 후 기존 확정 일정과 충돌 가능성이 생기면 트레이너에게 경고를 표시하되, 기존 일정은 유지합니다.
- **시간 설정 오류**: 종료 시간이 시작 시간보다 빠르거나 같으면 "유효하지 않은 시간 범위입니다"라는 안내를 표시합니다.

### 범위 외

- 회원이 직접 실시간 빈 슬롯을 보고 선점 예약하는 기능
- 동일 시간에 여러 명을 받는 오버부킹 설정
- AI 자동 최적 스케줄 추천 기능

## 9. PT 일정 배정 시스템

### 개요

회원이 매주 자신의 가능한 시간대를 등록하고, 트레이너가 회원별 가능 시간과 본인의 스케줄을 고려하여 일정을 직접 배정하는 시스템입니다. 회원은 배정된 일정을 확인하거나 변경을 요청할 수 있습니다.

기존의 예약 요청/승인/거절 구조 대신, **트레이너 주도 배정 및 회원 확인 구조**를 적용합니다.

### 사용자 흐름

#### 회원

- 홈 또는 일정 메뉴에서 [가능 시간 등록] 버튼을 누릅니다.
- 이번 주(또는 다음 주)의 가능한 시간대를 요일별로 선택합니다.
- 필요 시 메모를 입력합니다. (예: "목요일은 3시 이후만 가능")
- [전달하기] 버튼을 눌러 트레이너에게 전달합니다.
- 트레이너가 배정한 일정 알림을 확인합니다.
- [확인] 버튼을 눌러 일정을 확정하거나, [변경 요청] 버튼을 눌러 사유와 함께 변경을 요청합니다.

#### 트레이너

- 캘린더 또는 회원 관리 화면에서 회원별 가능 시간 등록 현황을 확인합니다.
- 미등록 회원에게는 리마인더 알림을 보낼 수 있습니다.
- 본인의 근무시간, 휴무일, 기존 확정 일정을 참고하여 빈 슬롯에 회원을 배정합니다.
- 회원의 가능 시간 등록 없이도 전화·대면·채팅으로 합의한 뒤 직접 배정할 수 있습니다.
- 회원의 변경 요청을 확인하고 다른 시간으로 재배정합니다.

### 화면 설명

1. **가능 시간 등록 화면 (회원용)**

   - 상단: 대상 기간 표시 (예: "3/17(월) ~ 3/23(일)")
   - 본문: 요일 × 시간대 다중 선택 그리드
     - 행: 월, 화, 수, 목, 금, 토, 일
     - 열: 오전 (06:00~12:00), 오후 (12:00~18:00), 저녁 (18:00~22:00)
     - 선택 방식: 토글 (탭하여 가능/불가능 전환)
   - 추가 입력: 메모 입력란 (선택사항)
   - 버튼: [전달하기]
   - 상태 표시: 이번 주 등록 완료 여부 표시

2. **일정 배정 확인 화면 (회원용)**

   - 정보: 트레이너 이름, 배정된 날짜/시간, 세션 단위
   - 버튼: [확인] (일정 확정), [변경 요청] (사유 입력 팝업)
   - 알림으로 진입 가능

3. **일정 배정 (트레이너용 — 캘린더 주간 뷰에서 직접 수행)**

   > 별도 배정 화면이 아닌, 캘린더 주간 뷰(섹션 10)가 배정 도구를 겸합니다.

   - 빈 슬롯 탭 시:
     - 회원 선택 바텀시트 표시
     - 각 회원 옆에 해당 시간대 가능 여부 아이콘 표시 (가능 ✅ / 불가 ❌ / 미확인 ❓)
     - 회원 선택 → 일정 생성 → 회원에게 알림
   - 기존 블록 탭 시:
     - 일정 상세 패널 → 취소, 재배정 등 액션

4. **회원 가능 시간 현황 (트레이너용)**

   - 리스트: 회원별 이번 주 가능 시간 요약
   - 상태: 등록 완료 / 미등록 배지
   - 미등록 회원에게 [리마인더 보내기] 버튼
   - 회원 메모 표시
   - 정렬: 미등록 회원 상단 노출

5. **변경 요청 팝업 (회원용)**

   - 입력: 변경 사유 선택 또는 자유 입력
   - 버튼: [요청 보내기], [닫기]

6. **변경 요청 처리 화면 (트레이너용)**

   - 정보: 회원 이름, 기존 배정 시간, 변경 사유
   - 액션: 새로운 시간 선택 → 재배정

### 비즈니스 규칙

- 회원은 직접 빈 슬롯을 선점하거나 예약 요청을 생성하지 않습니다.
- 회원은 매주 가능한 시간대를 등록합니다. 회원의 스케줄이 매주 달라지므로 주간 단위 등록이 기본입니다.
- 트레이너는 회원의 가능 시간, 본인의 근무시간, 기존 확정 일정을 참고하여 일정을 직접 배정합니다.
- 트레이너는 회원의 가능 시간 등록 없이도 직접 일정을 배정할 수 있습니다 (전화/대면/채팅 합의 후).
- 회원이 가능 시간을 등록하지 않으면 트레이너에게 미등록 상태로 표시되며, 리마인더 알림을 보낼 수 있습니다.
- 일정 배정 과정에서는 **거절** 상태를 사용하지 않고, 시간이 맞지 않으면 **변경 요청**으로 처리합니다.
- 확정된 일정은 시작 시간이 지나면 시스템이 자동으로 완료 상태로 변경합니다.
- 잔여 PT 횟수가 부족하더라도 일정 배정 자체는 가능하지만, 트레이너와 회원 모두에게 경고를 표시합니다.
- 취소는 PT 시작 전이라면 별도의 시간 제한 없이 트레이너 또는 회원 모두 가능합니다.

### 엣지 케이스

- **동일 시간 중복 배정**: 트레이너가 이미 배정/확정된 시간에 다른 회원을 배정하려 하면 저장을 차단하고 충돌 알림을 표시합니다.
- **변경 요청 후 재배정 전 취소**: 변경 요청 상태에서 트레이너가 재배정하기 전에 회원이 취소할 수 있습니다.
- **미등록 회원 배정**: 가능 시간 미등록 회원에게도 트레이너가 직접 배정할 수 있으며, 이 경우 회원의 가능 여부 아이콘은 "미확인"으로 표시됩니다.
- **휴무일 변경**: 배정 후 해당 날짜가 휴무일로 변경되면, 관련 일정은 트레이너에게 충돌 경고를 표시합니다. 기존 일정은 자동 취소되지 않으며 트레이너가 직접 처리합니다.

### 범위 외

- 회원 간 공개 선착순 예약 기능
- 대기 명단 자동 배정 기능
- 기존 확정 일정을 클릭 한 번으로 다른 시간으로 자동 이동시키는 기능
- 정기 PT 반복 일정 자동 등록 기능 (3차 개편 대상)

## 10. 캘린더

### 개요

트레이너와 회원이 전체 PT 일정을 달력 형태로 확인하는 기능입니다. **주간 타임라인 뷰가 기본(메인) 화면**이며, 월간 뷰는 날짜 이동 및 전체 현황 파악용 보조 화면으로 제공합니다.

트레이너에게는 일정 배정 도구이자 전체 스케줄 관리 화면이고, 회원에게는 자신의 배정 일정 확인 화면입니다.

> **주간 뷰가 메인인 이유**: PT 일정은 주 단위로 운영됩니다. 회원이 매주 가능 시간을 등록하고, 트레이너가 주간 단위로 배정합니다. 따라서 "이번 주 스케줄"을 한눈에 보는 주간 타임라인이 가장 자주 사용하는 화면이어야 합니다.

### 사용자 흐름

- 하단 네비게이션에서 캘린더 탭을 누르면 **이번 주 주간 타임라인**이 표시됩니다.
- 주간 뷰에서 시간대별로 배정/확정/변경요청 일정을 블록으로 확인합니다.
- 트레이너는 빈 슬롯을 탭하여 회원을 직접 배정할 수 있습니다.
- 상단의 [월간] 탭을 누르면 월간 뷰로 전환하여 전체 월의 일정 분포를 확인합니다.
- 월간 뷰에서 특정 날짜를 탭하면 해당 날짜가 포함된 주간 뷰로 이동합니다.

### 화면 설명

1. **주간 타임라인 뷰 (메인 화면)**

   - 상단: 주간 이동 화살표, 현재 주 표시 (예: "3/17 ~ 3/23"), [월간] 전환 탭
   - 구조:
     - 좌측 Y축: 시간축 (트레이너 근무 시작~종료 시간 기준, 예: 09:00~22:00)
     - 상단 X축: 요일 표시 (월~일, 오늘 날짜 강조)
   - 일정 블록 표시:
     - **배정됨**: 노랑 블록 (회원 이름, "확인 대기" 라벨)
     - **확정됨**: 초록 블록 (회원 이름, 시간)
     - **변경요청**: 주황 블록 (회원 이름, "변경요청" 라벨)
     - **완료**: 회색 블록
   - 빈 슬롯: 빈 칸 영역
   - 휴무일: 해당 요일 컬럼 전체 음영 처리
   - 비근무 시간대: 음영 처리 (근무시간 외 영역)
   - 트레이너 인터랙션:
     - 빈 슬롯 탭 → 회원 선택 바텀시트 → 배정
     - 일정 블록 탭 → 일정 상세 (상태 변경, 취소 등)
     - 변경요청 블록 탭 → 재배정 화면
   - 회원 인터랙션:
     - 배정됨 블록 탭 → [확인] / [변경 요청] 액션
     - 확정됨 블록 탭 → 일정 상세 (취소 가능)

2. **월간 뷰 (보조 화면)**

   - 상단: 월 이동 화살표, 현재 연월 표시, [주간] 전환 탭
   - 월간 그리드: 날짜 칸 안에 상태별 색상 점(도트) 표시
   - 상태 색상 (도트):
     - 배정됨 (회원 확인 대기): 노랑
     - 확정됨: 초록
     - 변경요청: 주황
     - 완료: 회색
     - 취소됨: 빨강
     - 휴무일: 배경 음영 처리
   - 날짜 탭 동작: 해당 날짜가 포함된 **주간 뷰로 전환** (날짜에 포커스)
   - 용도: 특정 주로 빠르게 이동, 월 전체 일정 밀도 파악, 휴무일 확인

3. **일정 상세 패널**

   - 진입: 주간 뷰에서 일정 블록 탭
   - 정보: 회원/트레이너 이름, 날짜, 시간, 상태, PT 잔여 횟수
   - 트레이너 액션:
     - 배정됨 → 취소 가능
     - 변경요청 → [재배정] 버튼 (시간 변경 바텀시트)
   - 회원 액션:
     - 배정됨 → [확인], [변경 요청] 버튼
     - 확정됨 → [취소] 버튼

### 비즈니스 규칙

- **주간 뷰가 기본 진입 화면**입니다. 캘린더 탭 진입 시 항상 이번 주 주간 뷰로 시작합니다.
- 월간 뷰는 날짜 탐색 및 전체 현황 파악용이며, 날짜 탭 시 주간 뷰로 돌아갑니다.
- 트레이너는 본인에게 연결된 모든 회원의 배정/확정 일정을 통합하여 확인합니다.
- 회원은 본인과 관련된 일정만 캘린더에서 볼 수 있습니다.
- 배정 일정과 확정 일정은 블록 색상으로 시각 구분합니다.
- 주간 뷰의 시간축 범위는 트레이너의 근무시간 설정에 따라 동적으로 조정됩니다.
- 지나간 일정도 완료 및 취소 이력으로 확인할 수 있습니다.
- 휴무일은 해당 요일 컬럼 전체가 음영 처리되며, 빈 슬롯 탭이 비활성화됩니다.

### 엣지 케이스

- **일정 없는 주**: 배정/확정 일정이 없으면 빈 타임라인에 "이번 주 일정이 없습니다" 메시지를 표시합니다.
- **변경 요청만 존재**: 확정 일정 없이 변경 요청 건만 있는 경우 주황 블록으로 눈에 띄게 표시합니다.
- **휴무일 표시**: 휴무일은 해당 컬럼 전체에 음영 처리하여 회원에게도 해당 날짜가 일반 운영일이 아님을 알립니다.
- **좁은 화면에서의 주간 뷰**: 480px 모바일 레이아웃에서 7일 컬럼은 좁을 수 있음. 좌우 스크롤 또는 5일(평일) 기본 표시 + 주말 스크롤 방식을 고려합니다.
- **오늘 표시**: 현재 날짜의 컬럼은 상단 요일 라벨에 강조 표시 (파란색 동그라미 등).

### 범위 외

- 외부 캘린더(구글, 애플 등) 연동
- 개인 To-do 기능
- OS 푸시 기반 리마인더 알림
- 일간 단독 뷰 (주간 뷰에서 충분히 확인 가능)

### 기존 테이블 변경

**reservations 테이블 — 상태 enum 변경:**

```
기존: 'pending', 'approved', 'rejected', 'cancelled', 'completed'
변경: 'scheduled', 'confirmed', 'change_requested', 'cancelled', 'completed'
```

**reservations 테이블 — 컬럼 변경:**

### 신규 테이블

**member_weekly_availability (회원 주간 가능 시간):**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| member_id | UUID | 회원 ID (FK → auth.users) |
| trainer_id | UUID | 트레이너 ID (FK → auth.users) |
| week_start | DATE | 주간 시작일 (월요일 기준) |
| available_slots | JSONB | 가능 시간 데이터 |
| memo | TEXT | 추가 메모 (선택) |
| created_at | TIMESTAMPTZ | 등록 시각 |

**available_slots JSONB 구조:**

```json
{
  "mon": ["morning", "afternoon"],
  "tue": ["afternoon", "evening"],
  "wed": [],
  "thu": ["morning"],
  "fri": ["afternoon", "evening"],
  "sat": [],
  "sun": []
}
```

- `morning`: 06:00~12:00
- `afternoon`: 12:00~18:00
- `evening`: 18:00~22:00
- 빈 배열: 해당 요일 불가

**유니크 제약조건:** `(member_id, trainer_id, week_start)` — 주간당 1건

### 트리거 변경

| 기존 트리거 | 변경 |
|------------|------|
| `trg_auto_deduct_pt` (완료 시 PT 차감) | 유지 — `completed` 상태 전이 시 동작 |

- 동작: reservations에 `scheduled` 상태로 INSERT + 알림 생성
- 반환: 일정 ID 또는 에러

## 기존 데이터 마이그레이션

### 상태값 매핑

| 기존 상태 | 마이그레이션 처리 |
|----------|----------------|
| `pending` | `scheduled`로 변환 (트레이너 배정으로 간주) |
| `approved` | `confirmed`로 변환 |
| `rejected` | 이력 보존, 조회 시 "변경됨"으로 표시 |
| `cancelled` | 그대로 유지 |
| `completed` | 그대로 유지 |

### enum 마이그레이션 순서

```sql
-- 1. 신규 값 추가
ALTER TYPE reservation_status ADD VALUE 'scheduled';
ALTER TYPE reservation_status ADD VALUE 'confirmed';
ALTER TYPE reservation_status ADD VALUE 'change_requested';

-- 2. 기존 데이터 변환
UPDATE reservations SET status = 'scheduled' WHERE status = 'pending';
UPDATE reservations SET status = 'confirmed' WHERE status = 'approved';

-- 3. rejected는 enum에서 삭제 불가 → 코드에서 사용하지 않도록 처리
-- PostgreSQL은 enum 값 삭제를 지원하지 않으므로 기존 값은 남겨두되 앱에서 제외
```

## 최종 방향

본 서비스는 일반적인 공개 예약 플랫폼이 아니라, **트레이너가 회원별 PT 일정을 효율적으로 배정하고 관리하는 운영형 서비스**를 지향합니다.

회원은 매주 변동되는 자신의 가능 시간대를 간편하게 전달하고, 트레이너는 이를 참고하여 최적의 시간을 배정합니다. 거절이나 경쟁 없이, **트레이너 주도 배정 → 회원 확인** 구조로 양측 모두에게 편리한 경험을 제공합니다.
