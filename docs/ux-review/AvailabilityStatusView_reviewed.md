## 최종 리뷰 내용

### Critical (즉시 수정 필요)

#### 1. 에러 상태가 빈 상태/미등록 상태로 오인됨
- **위치**: `AvailabilityStatusView.vue:264-265`, `AvailabilityStatusView.vue:317-328`
- **문제**: 회원 목록 조회가 실패하면 `members.value`가 빈 배열로 남아 "연결된 회원이 없습니다" 빈 상태로 표시된다. 가능 시간 조회가 실패하면 `availabilities.value`가 빈 배열로 남아 모든 회원이 미등록인 것처럼 보인다. 에러 토스트는 잠깐 뜨고 사라지므로 사용자가 놓칠 수 있다.
- **개선안**: `hasError` 분기를 추가하여 에러 상태를 명시적으로 표시하고, "다시 불러오기" 재시도 버튼을 제공한다.

#### 2. 히트맵 셀 터치 타겟 심각하게 부족
- **위치**: `AvailabilityStatusView.css:229-240`, `AvailabilityStatusView.css:183-187`
- **문제**: 히트맵 셀의 `min-height: 36px`로 높이가 최소 터치 타겟 44px에 미달한다. 480px 화면에서 시간 라벨(48px)을 빼면 각 셀 너비는 약 61px이므로 가로는 충분하지만, 112개 셀이 밀집된 상태에서 세로 36px은 오탭 가능성이 높다.
- **개선안**: `min-height`를 44px로 변경한다. 히트맵 전체 높이가 증가하지만 스크롤 가능 영역이므로 문제없다.

#### 3. 로딩 상태에서 스피너/스켈레톤 없음
- **위치**: `AvailabilityStatusView.vue:39-41`
- **문제**: 로딩 중에 텍스트만 표시된다. 다른 화면들은 `AppSkeleton`을 사용하고 있어 일관성이 부족하다.
- **개선안**: `AppSkeleton`으로 히트맵 형태를 모방하거나, 최소한 로딩 스피너를 추가한다.

### Major (높은 우선순위)

#### 4. 주간 네비게이션 화살표, 뒤로가기, 리마인더 버튼 터치 타겟 부족
- **위치**: `AvailabilityStatusView.css:92-108` (화살표 36px), `AvailabilityStatusView.css:23-34` (뒤로가기 40px), `AvailabilityStatusView.css:337-348` (리마인더 32px)
- **문제**: 주간 이동 버튼 36px, 뒤로가기 버튼 40px, 리마인더 버튼 32px으로 모두 최소 44px에 미달한다. 특히 리마인더 버튼은 바텀시트 내 핵심 액션인데 높이가 32px로 가장 작다.
- **개선안**: 모든 인터랙티브 버튼을 최소 44px hit area로 조정한다.

#### 5. 리마인더 전송 중 다른 회원 버튼도 비활성화
- **위치**: `AvailabilityStatusView.vue:122-123`, `AvailabilityStatusView.vue:341-343`
- **문제**: `reminderLoadingId`가 하나라도 설정되면 `Boolean(reminderLoadingId)`로 인해 모든 리마인더 버튼이 비활성화된다. 또한 `sendReminder` 함수 시작 부분의 guard도 `reminderLoadingId.value`가 truthy이면 즉시 return하여 다른 회원에게 전송을 차단한다.
- **개선안**: 동시 전송을 허용한다면 `reminderLoadingId`를 `Set` 기반으로 변경하여 개별 로딩 상태를 관리한다. 동시 전송을 제한한다면 현재 방식을 유지하되 UI에서 이유를 안내한다.

#### 6. 리마인더 문구가 주차와 무관하게 고정
- **위치**: `AvailabilityStatusView.vue:352-356`
- **문제**: 다음 주 화면에서도 리마인더 문구가 항상 "이번 주 PT 가능 시간을 등록해주세요"로 전송된다. 주간 전환 UX의 핵심 정보가 잘못 전달된다.
- **개선안**: `weekLabel` 또는 `weekRangeText`를 메시지 본문에 반영하여 "다음 주(3/24~3/30) PT 가능 시간을 등록해주세요" 형태로 변경한다.

#### 7. 0명 히트맵 셀 탭 시 dead-end 인터랙션
- **위치**: `AvailabilityStatusView.vue:72-81`, `AvailabilityStatusView.vue:90-91`
- **문제**: 0명 셀도 모두 버튼이라 탭하면 "가능한 회원이 없습니다" 바텀시트가 열린다. 112개 밀집 셀 그리드에서 무의미한 인터랙션을 유도한다.
- **개선안**: 0명 셀은 버튼이 아닌 정적 셀(`<div>`)로 렌더링하거나, `pointer-events: none`으로 비활성화한다.

#### 8. 가로 스크롤 힌트 부재 (조건부 발생)
- **위치**: `AvailabilityStatusView.css:173-186`
- **문제**: `overflow-x: auto`이지만 `min-width: 320px`이므로 일반적인 360px 이상 기기에서는 스크롤이 거의 발생하지 않는다. 매우 좁은 화면이나 글꼴 확대 시에만 문제가 된다.
- **개선안**: 우선순위를 낮추되, 그라데이션 페이드 효과를 오른쪽 끝에 추가하거나 `min-width`를 제거하여 항상 화면에 맞추도록 한다.

### Minor (개선 권장)

#### 9. 히트맵 셀에 aria-label 부재
- **위치**: `AvailabilityStatusView.vue:72-81`
- **문제**: 히트맵 셀 버튼에 `aria-label`이 없다. 스크린 리더 사용자가 셀의 의미를 파악할 수 없다.
- **개선안**: `:aria-label="\`${DAY_FULL_LABELS[day]} ${time} 가능 회원 ${heatmapData[day]?.[time]?.count ?? 0}명\`"`을 추가한다.

#### 10. 히트맵 0명 셀의 저대비 색상
- **위치**: `AvailabilityStatusView.css:247-250`
- **문제**: heat-0 셀의 텍스트 색상이 `var(--color-gray-200)`으로, 배경 `var(--color-gray-100)`과의 대비가 매우 낮아 WCAG AA 기준을 충족하지 못할 수 있다.
- **개선안**: 0명 셀 텍스트를 `var(--color-gray-400)` 이상으로 조정하거나, 0명일 때 텍스트를 숨기고 빈 셀로 표시한다.

#### 11. 미등록 회원 배너의 이모지 사용
- **위치**: `AvailabilityStatusView.vue:57`
- **문제**: "미등록" 표시에 이모지를 사용하여 OS/브라우저별 렌더링 차이가 있다.
- **개선안**: CSS로 빨간 점을 구현한다: `<span class="dot dot--red" />`

#### 12. 바텀시트 내 리마인더 전송 후 중복 전송 방지
- **위치**: `AvailabilityStatusView.vue:341-368`
- **문제**: 리마인더 전송 성공 시 토스트만 표시되고 해당 회원의 상태가 변경되지 않아 중복 전송 가능하다.
- **개선안**: 전송 완료된 회원의 버튼을 "전송 완료"로 변경하고 비활성화한다. `Set`으로 전송 완료 ID를 추적한다.

#### 13. 버튼 focus-visible 스타일 부재
- **위치**: `AvailabilityStatusView.css` 전반
- **문제**: 키보드 네비게이션 시 포커스 인디케이터가 표시되지 않는다.
- **개선안**: 인터랙티브 요소에 `:focus-visible` 스타일을 추가한다.

### Good (잘된 점)

- **빈 상태 처리**: "연결된 회원이 없습니다" + 부가 설명 텍스트로 빈 상태를 명확히 안내한다.
- **히트맵 색상 단계**: 4단계(0/1-2/3-5/6+) 히트 레벨이 직관적이고 파란색 계열로 일관되게 표현한다.
- **미등록 회원 배너**: 미등록 회원이 있을 때 상단에 눈에 띄는 배너를 표시하여 즉시 인지 가능하다.
- **race condition 방지**: `loadToken` 패턴으로 비동기 요청의 경쟁 조건을 방지한다.
- **sticky 헤더**: backdrop-filter blur가 적용된 sticky 헤더로 모던한 느낌을 준다.
- **바텀시트 활용**: 셀 클릭 시 가능 회원 목록, 미등록 회원 목록을 각각 바텀시트로 표시하여 화면 전환 없이 상세 정보 확인 가능하다.

### 토스 앱 참고 개선안

1. **풀투리프레시**: 데이터를 수동으로 새로고침하는 방법이 없다. 풀투리프레시를 지원하면 최신 데이터 확인이 편리하다.
2. **히트맵 위에 요약 정보**: "이번 주 등록률: 8/10명 (80%)" 같은 요약 수치를 히트맵 위에 표시하면 전체 현황을 한눈에 파악할 수 있다.
3. **일괄 리마인더**: 미등록 회원이 여러 명일 때 "전체 리마인더 보내기" 버튼이 있으면 편리하다.
4. **애니메이션 전환**: 이번 주/다음 주 전환 시 슬라이드 애니메이션으로 데이터 변경을 시각적으로 인지할 수 있다.

### 구조 개선 제안 (참고용)

1. **상수 분리**: `DAY_ORDER`, `DAY_LABELS`, `TIME_SLOTS` 등을 별도 유틸 파일로 분리하면 재사용 가능하다.
2. **히트맵 컴포넌트 분리**: 히트맵 그리드 렌더링 로직을 `AvailabilityHeatmap.vue` 컴포넌트로 분리하면 뷰 파일의 복잡도가 줄어든다.
